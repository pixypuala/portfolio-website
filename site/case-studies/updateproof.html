UpdateProof

A contract-driven WordPress compatibility and release-assurance engine with deterministic exit codes — so a risky update fails a check in CI instead of a live site.

  • Testing · CI
  • Deterministic exit codes
  • Status: active development

Problem

WordPress updates ship constantly — core, plugins, and themes all move independently. A release can quietly change behaviour that a site depends on, and the breakage often surfaces in production because nothing asserted the old guarantee still held. Teams are left choosing between updating blindly and never updating at all.

Approach

UpdateProof turns the guarantees a site relies on into explicit contracts, then verifies a candidate release against them. Each run resolves to a deterministic exit code, so the result is unambiguous and can gate a pipeline: the same inputs always produce the same verdict, and a broken contract stops the release rather than the users.

What is built

  • A contract runner that checks a candidate WordPress environment against declared compatibility expectations.
  • Deterministic exit codes designed for CI gating, so a failing contract blocks a merge or deploy predictably.
  • Fixtures that exercise passing and failing contracts, keeping the engine's own verdicts under test.

Boundaries

  • This is a reference engine in active development; the bundled contract set is a starting point, not exhaustive coverage of every plugin.
  • It verifies the contracts it is given — the assurance is only as strong as the expectations a team encodes.
  • No compatibility guarantees are claimed for arbitrary third-party plugins beyond what the contracts explicitly assert.
